# 项目 README 设计说明

## 目标

为项目根目录新增一份中文 `README.md`，同时承担 GitHub 项目展示和新用户快速上手两项职责。内容必须以当前仓库中已经实现并验证的功能为准，不使用虚假截图、无效下载链接或尚未完成的模型能力。

## 目标读者

- 希望快速了解项目用途和完成本地运行的使用者
- 准备进行 Android 二次开发或模型替换的开发者
- 需要了解手机端与 PC 局域网控制台协作方式的项目评审者

## 内容结构

1. 项目名称与一句话简介
2. 核心功能
3. 当前模型说明
4. 快速开始
5. 手机端与 PC 端使用方法
6. 自训练 YOLO 模型导入约束
7. 开发环境与构建、测试命令
8. 项目目录结构
9. 后续计划
10. 注意事项

## 关键事实

- 当前唯一内置模型为 `mobilenet_ssd.tflite`，对应 SSD MobileNet V1 通用目标检测模型。
- 手机端支持实时相机检测、阈值和推理硬件设置、巡检记录以及模型管理。
- PC 端控制台由手机内置 Web 服务提供，默认通过同一局域网的 `http://手机IP:8080` 访问。
- 模型页面保留一个自训练 YOLO TFLite 文件导入口。
- 导入模型需要兼容 TensorFlow Lite Task Vision `ObjectDetector`，并包含所需 Metadata；原始 Ultralytics TFLite 导出文件不保证可以直接加载。
- Windows 用户可以使用根目录的 `打包.bat` 和 `安装.bat`。
- 项目使用 Gradle Wrapper 9.3.1。普通构建可使用 JDK 17；基于 Android SDK 36 的 Robolectric 测试需要 Java 21 或更高版本。
- 最低 Android 版本为 Android 7.0（API 24）。

## 表达与展示原则

- README 以中文为主，命令、文件名和技术名词保留原文。
- 首页保持易读，优先提供可复制的命令和明确的操作顺序。
- 不放置当前仓库没有的 APK 下载地址、许可证声明、效果指标或模型精度数据。
- 暂不添加界面截图；待获得真实手机端和 PC 端截图后再增加“界面预览”章节。

## 验证方式

- 检查 README 中涉及的文件和命令在仓库中真实存在。
- 将模型名称、端口、最低 SDK、Gradle 和 Java 要求与代码及构建配置交叉核对。
- 检查 Markdown 标题层级、代码块和相对链接。
- 确认 README 不包含未完成标记、占位链接或未经实现的功能描述。
